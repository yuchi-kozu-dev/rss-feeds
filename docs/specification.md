# RSS Feed Reader - 仕様書（シンプル版）

## 1. プロジェクト概要

### 1.1 目的
設定ファイルに登録されたRSSフィードから記事を取得し、一覧表示するシンプルなWebアプリケーション。

### 1.2 ターゲットユーザー
- 複数のブログやニュースサイトを定期的にチェックしたいユーザー

### 1.3 成功基準
- 複数のRSSフィードから記事を取得できる
- 記事を時系列で表示できる
- レスポンス時間が5秒以内

---

## 2. 機能要件（最小限）

### 2.1 記事一覧表示

#### 2.1.1 記事取得
- **入力**: なし（設定ファイルからRSSリストを読み込む）
- **処理**: 
  1. 設定ファイルからRSSフィードURLリストを取得
  2. 各フィードから記事を取得
  3. 公開日時の降順でソート
- **出力**: 記事一覧

#### 2.1.2 記事一覧表示
- **表示項目**:
  - 記事タイトル
  - 公開日時
  - フィード名
  - 記事の概要（最初の150文字）
- **ソート**: 公開日時の降順（新しい順）
- **件数**: 全件表示（制限なし）

#### 2.1.3 記事リンク
- **動作**: 記事タイトルをクリックすると元サイトへ移動（新しいタブ）

### 2.2 更新機能

#### 2.2.1 手動更新
- **トリガー**: ユーザーの更新ボタンクリック
- **処理**: 全フィードの最新記事を再取得
- **フィードバック**: ローディング表示

## 4. データモデル

### 4.1 Article（記事）
```typescript
interface Article {
  title: string;           // 記事タイトル
  link: string;            // 記事URL
  description?: string;    // 記事概要
  pubDate: string;         // 公開日時（ISO 8601形式）
  feedTitle: string;       // フィード名
}
```

### 4.2 Feed設定
```typescript
// config/feeds.ts
export const RSS_FEEDS: string[] = [
  'https://example.com/feed.xml',
  // ...
];
```

## 5. API仕様

### 5.1 記事取得API

#### GET /api/articles
全フィードから記事を取得

**レスポンス（成功）**:
```json
{
  "success": true,
  "data": {
    "articles": [
      {
        "title": "記事タイトル",
        "link": "https://example.com/article",
        "description": "記事の概要...",
        "pubDate": "2025-11-23T00:00:00Z",
        "feedTitle": "Example Blog"
      }
    ],
    "totalCount": 50
  }
}
```

**レスポンス（エラー）**:
```json
{
  "success": false,
  "error": {
    "message": "記事の取得に失敗しました"
  }
}
```

## 9. 技術的制約

### 9.1 データ保存
- 記事データは保存しない（毎回取得）
- フィードリストは設定ファイルに記載（ハードコード）

### 9.2 スケーラビリティ
- 小規模利用を想定（フィード数: 最大10個）
- 大量のフィードには対応しない
